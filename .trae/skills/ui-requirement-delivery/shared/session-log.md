# Session Log — 会话过程记录规范

## 目的

在工作流执行过程中，将以下三类事件实时追加到独立日志文件，
保证决策过程可回溯、变更有据可查、重新生成的上下文不丢失。

---

## 触发记录的三类事件

| 类型 | 触发时机 | 标记 |
|---|---|---|
| `[CHANGE]` 用户修改 | 用户要求修改任何已生成的内容（需求、设计、代码、文档） | 🔧 |
| `[CONFIRM]` 确认项 | 用户对某个输出明确表示认可 / 锁定，后续不再变更 | ✅ |
| `[REGEN]` 重新生成 | 用户对输出不满意，要求重新生成（需记录原因和新方向） | 🔄 |

---

## 日志文件

- **文件名**：`workflow-session-log.md`
- **位置**：与其他交付物同目录（项目根或约定的文档目录）
- **创建时机**：工作流第一条记录产生时自动创建
- **追加方式**：每次事件后立即追加，不批量补写

---

## 单条记录格式

```markdown
## [类型标记] Phase X — 一句话事件标题

- **时间**：YYYY-MM-DD HH:mm（或「会话第 N 轮」）
- **Phase**：Phase X（阶段名称）
- **事件**：[CHANGE | CONFIRM | REGEN]

### 原内容（CHANGE / REGEN 必填，CONFIRM 可省略）
> 简要引用被修改或被否定的原始内容，关键字段即可，不需要全文

### 新方向 / 确认内容
> 用户的具体要求，或确认锁定的内容

### 影响范围（CHANGE / REGEN 必填）
- 受影响的文件 / REQ 编号 / 任务编号

---
```

```markdown

- **时间**：2024-03-15 14:32
- **Phase**：Phase 0（PRD 解析）
- **事件**：CHANGE

### 影响范围
- `prd-analyze.md`：REQ-006 优先级 P0 → P1
- `prd-dev-task-list.md`：FE-08 从 M1 移入 M2

## [CONFIRM] Phase 3 — 锁定品牌 Tab 复用 AssetTabs 组件

- **时间**：2024-03-15 15:10
- **Phase**：Phase 3（UI → 组件映射）
- **事件**：CONFIRM

### 确认内容
品牌切换 Tab 使用 `AssetTabs.tsx#18`，关闭右侧下拉菜单，不自定义开发

---

## [REGEN] Phase 5 — 重新生成 services/api.ts 骨架签名

- **时间**：2024-03-15 16:45
- **Phase**：Phase 5（代码骨架）
- **事件**：REGEN

---
```

## 各类事件的触发说明

### `[CHANGE]` 用户修改

以下行为均触发 CHANGE 记录：

- 修改 REQ 表中任何字段（描述 / 优先级 / 验收项）
- 调整组件选型结论（从复用改为自定义，或反之）
- 变更任务的 Sprint 归属 / 依赖关系
- 修改骨架中的文件结构、命名、职责边界
- 修改接口的字段名、类型、错误码映射

### `[CONFIRM]` 确认项

以下行为触发 CONFIRM 记录：

- 用户说「就这样」「确认」「锁定」「不用改了」「好的继续」
- 用户对某个 Phase 的输出表示认可，进入下一 Phase
- 用户批准某个设计决策（状态管理方案、分页策略、组件选型等）

### `[REGEN]` 重新生成

以下行为触发 REGEN 记录：

- 用户说「重新生成」「不对」「换个方式」「重来」「这个不是我想要的」
- 用户对输出结果表达明显不满，要求全部或部分重做
- AI 自动发现之前生成有误并主动重新生成

---

## 维护原则

- **实时追加，不延后**：每次事件发生后立即写入，不等 Phase 结束再补
- **简洁记录**：原内容只引用关键字段，不全文复制
- **不修改历史记录**：已写入的条目只追加不修改；若有补充，新建一条 CHANGE 记录
- **Session 结束时汇总**：在文件末尾追加一条 `[SUMMARY]` 条目

---

## SUMMARY 格式

```markdown
## [SUMMARY] 会话结束汇总

- **总记录数**：X 条（CHANGE: N / CONFIRM: N / REGEN: N）
- **最终锁定的关键决策**：
  1. （从 CONFIRM 条目中摘取）
  2. ...
- **累计 REGEN 最多的环节**：Phase X（N 次），原因：...
- **未解决的开放问题**：（若有）
```
