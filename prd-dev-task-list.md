# ICAR-4389 开发任务清单版（前端主导）

## 1. 目标与交付原则

- 目标：将「佣金汇率结算配置」PRD落地为可执行开发任务，支持排期、分工、联调和验收。
- 原则：先 P0 主链路上线（配置 + 统计），再做 P1 优化。
- 输出粒度：每项任务包含负责人角色、前置依赖、完成定义（DoD）。

## 4. 前后端联调清单

## 4.1 接口与字段联调
- 配置读取/保存：X/Y/Z 的空值语义、范围语义、精度语义统一。
- 历史记录：From/To 字段是否服务端直接返回格式化文本。
- 统计报表：行列口径、Total 计算边界、USD换算精度。
- 明细列表：总利润字段是否后端返回，或前端按行汇总。

## 4.2 错误码与提示映射
- 2FA 失败、过期、取消
- 输入超限、非法精度
- 无权限、数据为空、系统异常

## 4.3 时区与时间
- 页面统一时区口径
- 日期边界（00:00:00 / 23:59:59）与后端一致

## 5. 测试清单（可直接执行）

## 5.1 功能测试
- 三项汇率输入：空值、边界值、非法值、两位小数限制。

## 9. Harness 式执行与验收标准

### 9.1 任务执行闭环
- Step-1：任务领取时绑定 `REQ-*`
- Step-2：开发完成后提交对应 UT/E2E 用例编号
- Step-3：CI 通过后进入人工抽检
- Step-4：抽检通过才可进入灰度

### 9.2 最小门禁清单（建议写入 CI）
- 必过：lint
- 必过：typecheck
- 必过：单测（输入校验/权限/汇总）
- 必过：E2E 主链路（保存配置后统计展示一致）
- 必填：PR 中 `Implements: REQ-*` 与 `Tests: case-*`

### 9.3 失败处理机制
- 任一门禁失败：任务状态从 Done 回退到 In Progress
- 若 REQ 映射缺失：禁止进入测试阶段
- 若回归失败：仅允许修复回归问题，不允许顺带改需求范围
