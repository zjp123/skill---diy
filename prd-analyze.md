# ICAR-4389 佣金汇率结算配置 - 前端可执行需求拆解

## 1. 项目目标与范围

### 1.1 目标
- 支持全品牌佣金汇率扣点配置，覆盖跨币种、Pip_%、Per Million 三类汇率参数。
- 支持汇率利润统计与明细查询，形成按品牌、监管、日期维度的数据报表。
- 对 Admin 现有历史报表展示做兼容，展示扣点后的开仓点差和平仓价。

### 1.2 覆盖系统（前端关联）
- OWS：配置与统计主入口
- Admin：Trading history / Rebate history 展示兼容
- IB portal / Davinci / SWS：回归验证影响

### 1.3 非范围
- 暂不包含 CPA 佣金场景
- 白名单能力不做
- UI 由前端直接实现（无独立 UI 稿）

---

## 2. 前端实施总步骤（按执行顺序）

### Step 1：信息架构与路由落地
- 在 OWS「代理管理」下新增二级菜单：
  - 佣金汇率 - 汇率配置
  - 佣金汇率 - 汇率统计
- 补齐中英文菜单文案与页面标题文案。
- 按权限控制菜单可见性、页面可访问性、编辑按钮可见性。

### Step 2：汇率配置页（核心功能）实现
- 按品牌 Tab 展示配置与历史记录，Tab 顺序：VG, VT, PU, MM, UM, VTJ, ST。
- 实现三项汇率配置输入（0.01%~10.00%，支持清空，最多 2 位小数，不可负数）。
- 实现编辑/保存切换与保存成功提示；保存前触发谷歌两步验证流程。
- 实现历史记录列表（From / To / 操作者 / 时间）。

### Step 3：汇率统计页实现
- 实现利润统计表（按返佣类型列、按扣点模式行、总计与佣金行）。
- 实现利润明细查询（监管、佣金日期）和明细列表字段展示。
- 支持默认最近 30 天、最长 366 天（统计）与最长 30 天（明细）的日期筛选策略。

### Step 4：Admin 展示兼容
- 在 Trading history、Rebate history 的开仓点差与平仓价列展示扣点后结果值。
- 保持历史页面交互不变，仅替换字段值来源/显示逻辑。

### Step 5：跨系统回归与灰度
- 回归 IB 佣金、销售佣金、PAMM、跟单返佣、IB 分佣、监控、重算、API 同步等链路。
- 按品牌分批灰度，先小品牌观察后全品牌放开。

## 3. 模块拆解（功能点 / 前端任务 / 验收标准）

## 3.1 模块 A：OWS 菜单与权限

### 功能点
- 菜单入口新增（配置、统计）
- 菜单权限控制（无权限不可见）
- 功能权限控制（无权限无编辑按钮）

### 前端任务
- 路由注册与国际化 key 注入
- 权限点接入（menu-level + action-level）
- 未授权场景兜底页/空态提示

### 验收标准
- 有权限用户可见菜单并可访问页面
- 无权限用户菜单不可见且无法通过 URL 越权访问
- 编辑按钮与权限实时一致

---

## 3.2 模块 B：汇率配置页

### 功能点
- 品牌 Tab 切换 + 各品牌独立配置
- 三个配置项：
  - 跨币种汇率（X）
  - Pip_% 汇率（Y）
  - Per Million 汇率（Z）
- 每个配置项带问号说明提示
- 编辑态与保存态切换
- 保存成功提示（中/英）

### 前端任务
- 页面结构：配置区 + 历史记录区
- 表单校验：
  - 可为空
  - 非空时范围 0.01~10.00
  - 最多 2 位小数
  - 非法提示：汇率要求0.01～10.00 %内，请重新输入
- 输入限制：超出精度直接拦截
- 保存流程串联 2FA 验证

### 验收标准
- 校验规则与 PRD 一致，边界值可测
- 保存后立即刷新为最新值并产生一条历史记录
- 保存按钮在请求中防重提交

---

## 3.3 模块 C：历史记录

### 功能点
- 标题：{品牌名}+历史记录
- 每次保存成功新增一条记录
- 字段：
  - From：上一次值（默认“-”）
  - To：新值
  - 操作者：用户名称
  - 时间：yyyy-MM-dd HH:mm:ss

### 前端任务
- 组合展示格式：{汇率类型}：{值%}
- 空值与首条记录兼容
- 品牌切换后历史记录联动刷新

### 验收标准
- 记录顺序最新在前
- From/To 展示语义准确
- 时间格式统一

---

## 3.4 模块 D：汇率统计（汇率利润）

### 功能点
- 筛选条件：
  - 监管（多选）
  - 日期（默认最近30天，最长366天）
- 指标：
  - 利润 = 未扣点佣金 - 扣点后佣金
  - 佣金 = 实际下发佣金额
- 表格维度：
  - 列：7 类返佣类型 + Total
  - 行：Cross Currency / Pip_% / Per Million + 总计行 + 实际下发行

### 前端任务
- 复杂表头与汇总行渲染
- USD 维度展示格式化（精度、千分位）
- 品牌维度切换与全品牌总计展示

### 验收标准
- 统计口径与后端一致
- 筛选变化触发重算并刷新表格
- Total 行列加总正确

---

## 3.5 模块 E：利润明细

### 功能点
- 筛选条件：
  - 监管（单选）
  - 佣金日期（默认最近30天，最长30天）
- 列表字段：
  - 品牌、监管、日期
  - 跨币种汇率/利润
  - Pip_% 汇率/利润
  - Per Million 汇率/利润
  - 总利润(USD)
  - 佣金(USD)
- 列表左下角显示 Total Profit（当前列表总计）

### 前端任务
- 查询表单与列表联动
- 列字段格式化与空值处理
- 底部总计展示与分页同步逻辑

### 验收标准
- 条件过滤准确，支持精确定位记录
- 总利润与列表内容一致
- 日期范围限制生效

---

## 3.6 模块 F：Admin 报表兼容

### 功能点
- 开仓点差展示扣点后结果
- 平仓价展示扣点后结果
- 页面：
  - 账户管理-交易账号-Trading history
  - 账户管理-返佣账号-Rebate history

### 前端任务
- 替换字段映射，不改变现有交互
- 与原筛选、排序、分页能力兼容

### 验收标准
- 同一记录在相关系统口径一致
- 无历史功能回归问题

---

## 3.7 模块 G：国际化文案

### 功能点
- 菜单、标题、按钮、提示、表头全量中英文化

### 前端任务
- 补齐 i18n key
- 保证中英文文案长度下布局不破坏

### 验收标准
- 中英文切换后页面无缺失 key
- 提示文案与 PRD 语义一致

## 4. 前端关键规则沉淀

## 4.1 输入与校验规则
- 统一百分比输入组件，支持 `0.01~10.00`、2 位小数、可清空。
- 非法输入优先前端拦截，后端错误码二次兜底展示。

## 4.2 状态管理规则
- 品牌维度、筛选条件、分页条件进入 URL query，支持分享与回退。
- 编辑态离开页面保护（未保存提示）。

## 4.3 安全与合规
- 保存动作必须串联二次验证 token。
- 权限校验采用前后端双重校验，不在前端暴露敏感计算逻辑。

---

## 5. 接口契约清单（前端需对齐后端）

## 5.1 汇率配置类
- 获取品牌配置：按品牌返回 X/Y/Z 与更新时间、操作者
- 保存品牌配置：提交 X/Y/Z + 2FA 凭证
- 获取历史记录：按品牌分页返回 From/To/操作者/时间

## 5.2 统计类
- 获取汇率利润统计：品牌 + 监管(多选) + 日期区间
- 获取利润明细：品牌 + 监管(单选) + 日期区间 + 分页

## 5.3 Admin 展示类
- Trading/Rebate history 返回扣点后开仓点差、平仓价字段

## 5.4 通用约束
- 统一金额与汇率精度
- 统一时区与时间格式
- 统一错误码到前端提示映射

## 8. 风险与前端应对

- 风险：计算口径多、字段命名易混淆  
  应对：前后端对齐字段字典，联调前冻结口径表。

- 风险：统计表复杂，合计逻辑易出错  
  应对：前端加快照测试 + 固定样例数据回归。

- 风险：无 UI 稿，交互细节偏差  
  应对：先出低保真页面定义并与产品快速确认。

- 风险：多系统联动回归面大  
  应对：建立跨系统回归清单，按品牌灰度发布。

## 10. 验收门禁模板（Harness/CI 可执行）

### 10.1 PR 合并前门禁
- Gate-1：`REQ` 覆盖率 = 100%（每个 REQ 至少映射一个任务与一个验收项）
- Gate-2：lint/typecheck 通过
- Gate-3：核心单测通过（输入校验、权限渲染、汇总逻辑）
- Gate-4：主链路 E2E 通过（配置保存→统计→Admin 展示）
- Gate-5：高风险项人工抽检通过（权限、2FA、金额精度、状态机）

### 10.2 发布前门禁
- Gate-6：回归清单通过（IB/Davinci/SWS/Admin）
- Gate-7：灰度品牌观测通过（错误率、保存成功率）
- Gate-8：回滚开关验证通过

### 10.3 审计留痕
- 每个 PR 描述必须包含：
  - `Implements: REQ-xxx`
  - `Tests: UT/E2E case id`
  - `Risk: high/medium/low`
